# Релизный Чеклист

1. Проверить staged files.
2. Убедиться, что secrets и runtime state не попали в stage.
3. Скомпилировать переводы.
4. Запустить тесты.
5. Обновить `VERSION`.
6. Обновить `CHANGELOG.md`.
7. Сделать commit.
8. Создать tag.
9. Запушить branch и tags.
10. Опубликовать GitHub Release.
11. Синхронизировать GitHub Wiki из `docs/wiki`.
12. Задеплоить контейнер.
13. Проверить логи и smoke tests.
