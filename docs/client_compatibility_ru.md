# Совместимость клиентов

## Happ

- Основная подписка выдается как `text/plain; charset=utf-8` со строками конфигураций.
- Пользовательский путь в боте рассчитан на Happ для iOS, Android и Windows.
- Режим **РФ-сервисы напрямую, остальное через VPN** выдается отдельной Happ-настройкой. Основная подписка и этот режим не смешиваются.
- Подписка обхода БС выдается отдельными кнопками:
  - рекомендуемый вариант: `/wl-filtered/{vpn_id}`;
  - запасной вариант: `/wl/{vpn_id}`.

## Другие клиенты

- Активный пользовательский onboarding не предлагает выбор альтернативных клиентов.
- Технический raw-режим `format=raw` сохранен в прокси основного профиля для совместимости и диагностики, но не показывается отдельной кнопкой в боте.
- Happ routing rules не выдаются как универсальный формат для сторонних клиентов.

## Цвет ping и `n/a`

Цвет ping/`n/a` задается интерфейсом клиентского приложения. Серверная подписка не управляет цветом этих элементов в Happ.

Обходное решение на стороне проекта:

- не отдавать заведомо пустые или HTML-ответы вместо подписки;
- сортировать доступные узлы по последней измеренной задержке;
- добавлять текстовые маркеры качества `[OK]` и `[SLOW]` в названия узлов, чтобы пользователь не зависел только от цвета ping.
